# 基于 RAG 与 Agent 的智能面试辅导系统

> 一个面向求职面试场景的智能辅导系统，融合 **RAG（检索增强生成）**、**Agent（工具调用）**、**意图分类** 与 **状态机编排**，支持问答、模拟面试、岗位定制出题、用户状态保存与面试报告生成。

## 项目简介

本项目使用 Streamlit 构建交互界面，核心目标是帮助用户进行面试准备与模拟练习。系统会先从知识库中检索相关内容，再结合大模型与工具能力生成更贴近真实场景的回答或面试问题。

当前项目已经具备以下能力：

- **问答模式**：针对用户提问，基于知识库进行回答
- **模拟面试模式**：扮演面试官，进行连续追问与引导
- **岗位定制**：面试前先选择岗位，再根据岗位输出更专业的问题
- **意图分类**：识别用户是在回答问题、请求提示、闲聊打断还是想结束面试
- **状态机编排**：控制是否继续追问、切换下一题、给提示或结束面试
- **RAG 检索增强**：结合向量召回、多路召回、关键词召回与重排序
- **Self-RAG 保护**：当知识不足时，降低胡编风险
- **流式输出**：问答与模拟面试支持逐步输出，更接近真实聊天体验
- **用户状态持久化**：按用户 ID 保存历史记录、面试状态与报告内容

## 项目亮点

- **双模式交互**：支持 `问答模式` 和 `模拟面试模式`
- **岗位驱动出题**：面试题更贴近岗位要求，而不是随机泛化提问
- **主动引导能力**：用户回答较弱时会给提示，而不是直接切下一题
- **上下文连续性**：保留历史记录，可继续上次面试
- **可扩展性强**：后续可继续扩展语音、图片、简历解析等多模态能力

## 核心功能

### 1. 问答模式
用户输入问题后，系统会先检索知识库，再结合模型生成回答。适合快速查阅技术概念、项目细节、配置说明和岗位相关知识点。

### 2. 模拟面试模式
用户先选择岗位，系统会从对应岗位题库中生成第一题，并根据用户回答不断追问、提示或切题，形成更接近真实面试的流程。

### 3. 面试报告
面试结束后，系统会汇总完整对话记录和知识库参考内容，生成一份面试总结报告，帮助用户复盘。

### 4. 工具调用
系统支持以下工具能力：
- 获取当前城市
- 查询天气
- 获取当前用户 ID
- 获取当前月份
- 从外部 CSV 中读取用户记录

### 5. 检索优化
检索部分支持：
- 查询改写
- 多路召回
- 关键词 / BM25-like 召回
- 重排序
- Self-RAG 兜底

## 技术栈

- **Python 3.10+**
- **Streamlit**：前端交互界面
- **LangChain / LangGraph**：Agent 与链式调用
- **ChromaDB**：向量数据库
- **DashScope / Tongyi**：大模型与嵌入模型
- **RAG**：检索增强生成
- **高德开放平台 API**：城市定位与天气查询

## 快速开始

### 1. 克隆仓库
```bash
git clone <your-repo-url>
cd <your-repo-name>
```

### 2. 创建虚拟环境（推荐）
```bash
python -m venv .venv
.venv\Scripts\activate
```

### 3. 安装依赖
```bash
pip install -r requirements.txt
```

### 4. 配置 API 与模型参数
请根据你的实际环境修改以下配置文件：

#### `config/agent.yml`
- `amap_key`：高德地图 API Key
- `external_data_path`：外部用户记录文件路径

#### `config/rag.yml`
- `chat_model_name`：对话模型名称
- `embedding_model_name`：嵌入模型名称
- `rerank_model_name`：重排序模型名称
- `enable_query_rewrite`：是否启用查询改写
- `enable_multi_recall`：是否启用多路召回
- `enable_rerank`：是否启用重排序

#### `config/chroma.yml`
- `persist_directory`：向量数据库持久化目录
- `data_path`：知识库文件目录
- `k`：最终返回文档数

如果你使用的是本地模型或其他云服务，还需要补充相应的环境变量或连接配置。

### 5. 启动项目
```bash
streamlit run app.py
```

## 使用说明

### 第一步：加载知识库
首次运行建议在左侧点击 **加载/更新知识库**，系统会扫描 `data` 中允许的文档并构建向量索引。

### 第二步：切换用户
输入用户 ID 后点击 **切换/加载用户**，系统会自动恢复该用户的历史对话、面试状态与报告内容。

### 第三步：选择模式
- **问答模式**：适合单轮问题解答
- **模拟面试模式**：适合连续追问与岗位化面试训练

### 第四步：生成报告
在模拟面试结束后，可勾选生成报告并一键生成本次面试总结。

## 数据说明

- `data/user_histories/`：保存用户状态数据
- `data/` 下的知识库文件：用于 RAG 检索
- 向量库与索引目录：由配置文件中的 `persist_directory` 决定
